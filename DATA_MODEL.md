# DATA MODEL — Mosquito Trap Dashboard (Star Schema)\n\n## 1) Dimension: dim_date\n- DateKey (PK)\n- Date\n- Year\n- Quarter\n- Month\n- MonthName\n- WeekOfYear\n- Day\n- DayOfWeek\n\n## 2) Dimension: dim_area\n- AreaKey (PK)\n- area_id\n- area_name\n- district\n- ward\n- latitude\n- longitude\n- risk_level (low/medium/high)\n\n## 3) Dimension: dim_trap\n- TrapKey (PK)\n- trap_id\n- trap_code\n- AreaKey (FK -> dim_area)\n- install_date\n- trap_type\n- current_status\n- last_maintenance_at\n- battery_level\n\n## 4) Dimension: dim_status\n- StatusKey (PK)\n- status_name\n\n## 5) Fact: fact_observation\n- ObservationKey (PK)\n- DateKey (FK -> dim_date)\n- AreaKey (FK -> dim_area)\n- TrapKey (FK -> dim_trap)\n- observed_at (timestamp)\n- mosquito_count\n- species (optional)\n- temperature_c (optional)\n- humidity_pct (optional)\n- rainfall_mm (optional)\n\n## 6) Fact: fact_trap_health\n- HealthKey (PK)\n- DateKey (FK -> dim_date)\n- AreaKey (FK -> dim_area)\n- TrapKey (FK -> dim_trap)\n- StatusKey (FK -> dim_status)\n- check_time\n- battery_level\n- signal_strength (newly added in ETL)\n- error_code (newly added in ETL)\n\n## 7) Fact: fact_alert\n- AlertKey (PK)\n- DateKey (FK -> dim_date)\n- AreaKey (FK -> dim_area)\n- TrapKey (FK -> dim_trap)\n- alert_time\n- alert_type (offline/spike/maintenance_due)\n- severity (low/medium/high/critical)\n- resolved (boolean)\n- resolved_at (nullable)\n- message\n\n## 8) Quan hệ\n- dim_date 1—N fact_observation\n- dim_date 1—N fact_trap_health\n- dim_date 1—N fact_alert\n- dim_area 1—N dim_trap\n- dim_area 1—N fact_observation\n- dim_area 1—N fact_trap_health\n- dim_area 1—N fact_alert\n- dim_trap 1—N fact_observation\n- dim_trap 1—N fact_trap_health\n- dim_trap 1—N fact_alert\n- dim_status 1—N fact_trap_health\n\n## 9) Chỉ số tính toán mẫu\n- Trap Uptime (%) = số bản ghi health status=active / tổng bản ghi health\n- Avg Mosquito per Active Trap = tổng mosquito_count / số active traps\n- Spike Alert = mosquito_count hiện tại > trung bình 7 ngày * hệ số (ví dụ 1.8)\n
